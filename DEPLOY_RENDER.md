# Публикация в интернет через Render

Этот вариант подходит для тестового бесплатного запуска на несколько недель:

- одна публичная ссылка;
- автоматическое переключение `desktop / mobile` по устройству;
- HTTPS от Render;
- пользователям не нужна регистрация.

## Что уже подготовлено в проекте

- единый сервер [server.js](</Users/kirillsevcenko/Desktop/Пост мейкер/server.js>) для обеих версий;
- маршруты:
  - `/` - автоопределение устройства;
  - `/desktop/` - desktop-версия;
  - `/mobile/` - мобильная версия;
- blueprint [render.yaml](</Users/kirillsevcenko/Desktop/Пост мейкер/render.yaml>) для Render;
- health check: `/api/config`.

## Перед публикацией

1. Убедитесь, что локально все работает.
2. Проверьте, что `.env` не попадает в репозиторий.
3. Загрузите проект в GitHub-репозиторий.

## Пошаговый деплой

1. Откройте [Render Dashboard](https://dashboard.render.com/).
2. Нажмите `New`.
3. Выберите `Blueprint`.
4. Подключите GitHub-репозиторий с этим проектом.
5. Render увидит файл `render.yaml` и предложит создать web service.
6. Подтвердите настройки.
7. После создания сервиса откройте `Environment`.
8. Добавьте секрет `GEMINI_API_KEY`.
9. Если нужно, измените `GEMINI_IMAGE_MODEL` или `GEMINI_TEXT_MODEL`.
10. Нажмите `Manual Deploy` -> `Deploy latest commit`, если Render не перезапустил сервис сам после добавления ключа.
11. Дождитесь статуса `Live`.
12. Откройте выданный адрес вида `https://post-maker-shtunda13.onrender.com`.

## Как это будет работать для людей

- если человек откроет ссылку с телефона, сервер отправит его на `/mobile/`;
- если человек откроет ссылку с компьютера, сервер отправит его на `/desktop/`;
- если нужно вручную зафиксировать версию:
  - `https://...onrender.com/?view=mobile`
  - `https://...onrender.com/?view=desktop`

## QR-код

Для QR-кода используй корневой адрес:

- `https://ваш-сервис.onrender.com`

Сервер сам решит, какую версию открыть.

## Если захочешь свой домен

1. Открой сервис в Render.
2. Перейди в `Settings` -> `Custom Domains`.
3. Добавь домен.
4. Настрой DNS у регистратора.
5. Подтверди домен в Render.

## Что важно знать о бесплатном тарифе

- бесплатный web service может “засыпать” после простоя;
- первое открытие после сна может быть медленным;
- для тестового этапа это нормально, но для постоянного боевого запуска лучше потом перейти на более стабильный вариант.
